# 第十九章：内置命令详解

## 概要

本章深入分析了 Claude Code 的内置命令系统，揭示了三种命令执行模式（prompt、local、local-jsx）的设计原理和实现机制。命令系统是用户与 Claude Code 交互的核心入口，通过斜杠命令触发各种自动化工作流。

## 核心要点

### 命令类型系统

Claude Code 的命令分为三种执行模式：

| 类型 | 执行方式 | 适用场景 | 示例 |
|------|----------|----------|------|
| `prompt` | 展开为提示文本发送给模型 | 需要模型智能处理的任务 | `/init`, `/commit`, `/review` |
| `local` | 本地同步执行返回结果 | 纯本地操作、无需 UI | `/compact` |
| `local-jsx` | 渲染 React/Ink UI 界面 | 需要用户交互的选择界面 | `/config`, `/mcp`, `/memory` |

### 关键命令分析

**`/init` 命令**：采用八阶段工作流进行项目初始化，包括确定配置范围、探索代码库、获取补充信息、写入 CLAUDE.md 等步骤。新版提示强调"每行都必须通过测试：删除这行是否会导致 Claude 犯错？"

**`/commit` 命令**：使用工具白名单限制可用工具为 `Bash(git add:*)`, `Bash(git status:*)`, `Bash(git commit:*)`，通过 `!\`command\`` 语法预执行 shell 命令获取实时 git 状态。

**`/compact` 命令**：支持三种压缩策略层次——session memory compaction（最高）、reactive compact（中等）、traditional compact（最低）。

### 命令注册与加载机制

命令在 `src/commands.ts` 中集中注册，加载层次为：bundledSkills → builtinPluginSkills → skillDirCommands → workflowCommands → pluginCommands → pluginSkills → COMMANDS()（内置命令优先级最低）。

## 源文件位置

- `src/types/command.ts`：命令类型定义
- `src/commands.ts`：命令注册和查找
- `src/commands/init.ts`：`/init` 项目初始化
- `src/commands/commit/index.ts`：`/commit` 提交自动化
- `src/commands/review.ts`：`/review` PR 审查
- `src/commands/compact/compact.ts`：`/compact` 上下文压缩
- `src/commands/config/config.tsx`：`/config` 配置管理
- `src/commands/mcp/mcp.tsx`：`/mcp` MCP 管理
- `src/commands/memory/memory.tsx`：`/memory` 记忆系统
