# BashTool 深度解析

BashTool 是 Claude Code 中使用频率最高的工具，负责执行 Shell 命令，扮演"执行引擎"角色。

## 核心内容

### 执行流程

采用 AsyncGenerator 模式：
- 入口函数处理模拟 sed 编辑、输出累积
- runShellCommand Generator 实现实时进度、可控中断、后台切换
- 进度信号由共享轮询器触发

### Sandbox 安全机制

双重防护策略（权限检查 + 沙箱隔离）：
- `shouldUseSandbox()` 判断沙箱启用
- `containsExcludedCommand()` 检测排除命令
- SandboxManager 适配层：支持 macOS（seatbelt）、Linux/WSL2（bubblewrap）
- 配置转换：文件系统路径、网络域名、忽略违规

### 权限检查流程

多层防护，从精确匹配到语义检查：
- 第一层：精确匹配检查
- 第二层：规则匹配（Deny/Ask/Allow）
- 第三层：路径约束检查
- 第四层：模式检查
- 第五层：安全检查（危险模式检测）

关键安全设计：
- 复合命令防护：前缀规则不能匹配复合命令
- 环境变量剥离：deny/ask 规需剥离所有环境变量
- 安全包装器剥离：允许匹配带 timeout/time 的命令

### 进度追踪与输出处理

- EndTruncatingAccumulator 保留末尾内容
- 大输出持久化到磁盘（最大 64MB）
- UI 进度渲染组件

### 后台执行模式

三种触发方式：显式请求、自动超时（15s）、用户手动（Ctrl+B）
- Kairos 模式下自动后台化阻塞命令
- 后台任务结果包含任务 ID 和输出路径

## 设计启示

- 双重安全边界确保信任最小化
- AsyncGenerator 提供实时反馈
- 优雅降级处理大输出和长任务
- 智能规则建议提升用户体验
