# 第二十章：技能系统架构

## 概要

技能系统是 Claude Code 提供专业化能力的核心机制。通过技能，用户可以扩展 AI 能力边界、定义自动化流程、定制模型行为、隔离执行上下文。本章深入分析了技能的定义、加载、解析与执行机制。

## 核心要点

### 技能文件结构

技能以 Markdown 文件形式定义，推荐使用目录格式：`.claude/skills/my-skill/SKILL.md`。单文件格式（`.claude/commands/my-command.md`）作为 legacy 仍支持但已不推荐。

### Frontmatter 字段详解

技能通过 YAML Frontmatter 定义元数据，核心字段包括：

- `name`：显示名称（默认取目录名）
- `description`：技能描述
- `allowed-tools`：允许使用的工具列表
- `model`：专属模型（"inherit" 表示继承默认）
- `context`：执行上下文类型（`inline` 或 `fork`）
- `effort`：努力级别（low/medium/high 或整数）
- `paths`：路径匹配模式（条件技能）
- `hooks`：注册的钩子配置

### 多源加载策略

技能加载按优先级分层：

1. Managed Skills：企业策略目录
2. User Skills：用户全局配置目录（`~/.claude/skills`）
3. Project Skills：项目目录（`.claude/skills`）
4. Additional Dirs：通过 `--add-dir` 指定
5. Legacy Commands：旧版 `.claude/commands` 目录
6. Bundled Skills：CLI 内置技能

### Fork Context 执行机制

`context: 'fork'` 时技能在独立子 Agent 中执行，使用 `agent` 字段指定 Agent 类型（如 `general-purpose`、`Bash` 或自定义类型）。

### 条件技能与动态发现

技能可通过 `paths` 字段定义为条件技能，仅在匹配文件被操作时激活。系统通过向上遍历目录发现沿途的技能目录，使用 `ignore` 库进行 gitignore 风格的路径匹配。

## 源文件位置

- `src/skills/loadSkillsDir.ts`：技能加载核心逻辑
- `src/skills/bundledSkills.ts`：内置技能注册
- `src/types/command.ts`：Command 类型定义
- `schemas/hooks.ts`：Hooks Schema 定义
