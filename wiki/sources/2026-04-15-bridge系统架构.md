Bridge 系统是 Claude Code 远程控制功能的核心实现，允许用户通过 claude.ai/code 网站或 Claude 移动应用控制本地运行的会话。

## 设计理念

建立持久化服务端点，作为本地环境与云端服务的桥梁。实现"本地执行、远程控制"的创新模式。

## 核心场景

1. 跨设备连续性：办公室启动的任务可在家中通过手机继续
2. 协作共享：团队成员可查看他人正在运行的会话进度
3. 移动办公：远程审批权限请求无需 SSH 连接

## 架构原则

- 本地优先执行：敏感操作在本地执行
- 云端控制面：用户交互界面托管在云端
- 会话隔离：git worktree 或目录隔离机制
- 优雅降级：网络中断时本地会话继续运行

## 会话启动模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| single-session | 单会话，结束时 Bridge 退出 | 传统远程控制 |
| same-dir | 多会话共享同一工作目录 | 并发任务执行 |
| worktree | 每个会话在独立 git worktree 运行 | 需隔离的并行开发 |

## 传输层

v1 WebSocket 方案：
- 本地开发环境使用 ws:// 协议直连
- 生产环境使用 wss:// 通过 Envoy 代理

v2 SSE 方案：
- 序列号支持断点续传
- 心跳机制保持连接活跃
- 状态上报支持

## Work 消息类型

| 类型 | 处理逻辑 |
|------|----------|
| session | 启动新的本地会话进程 |
| healthcheck | 健康检查，确认 Bridge 活跃 |

## 心跳机制

关键作用：
- 延长租约防止服务端超时回收
- 认证检测触发 JWT 过期重新获取
- 健康监控持续验证会话可用性

## 优雅关闭流程

1. 终止子进程：SIGTERM → SIGKILL
2. 清理 Worktree
3. 停止 Work 通知服务端
4. 归档会话
5. 注销环境
