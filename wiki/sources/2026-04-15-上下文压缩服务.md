Claude Code 的上下文压缩服务是为解决长对话历史超出 API 上下文窗口限制而设计的核心模块。采用多层渐进式策略，从轻量级到重量级依次尝试。

## 核心目标

- 释放上下文空间：通过摘要和裁剪减少 token 占用
- 保留关键信息：确保技术细节、决策和进度不丢失
- 保证缓存效率：利用 Prompt Cache 减少重复计算
- 用户友好：提供自动和手动两种压缩方式

## 压缩层级

| 层级 | 模块 | 功能 |
|------|------|------|
| 微压缩 | microCompact.ts | 清理旧工具结果，轻量级预处理 |
| 自动压缩 | autoCompact.ts | 监控阈值，触发压缩流程 |
| Session Memory 压缩 | sessionMemoryCompact.ts | 使用预提取会话记忆替代 API |
| 主压缩 | compact.ts | 调用 Claude API 生成结构化摘要 |

## 触发条件

三种触发方式：
1. **自动触发**: token 使用量达到阈值（约 200K - 13K buffer）
2. **时间触发**: 距离上次助手消息超过 API 缓存 TTL（默认 60 分钟）
3. **手动触发**: 用户执行 `/compact` 命令

## Token 预算管理

关键常量：
- AUTOCOMPACT_BUFFER_TOKENS: 13,000
- WARNING_THRESHOLD_BUFFER_TOKENS: 20,000
- ERROR_THRESHOLD_BUFFER_TOKENS: 20,000
- MANUAL_COMPACT_BUFFER_TOKENS: 3,000

多级阈值判断：警告 → 错误 → 阻塞限制

## 摘要策略

结构化摘要模板包含 9 个部分：
1. Primary Request and Intent
2. Key Technical Concepts
3. Files and Code Sections
4. Errors and fixes
5. Problem Solving
6. All user messages
7. Pending Tasks
8. Current Work
9. Optional Next Step

## 工具结果截断

可压缩工具类型：FileRead, Shell, Grep, Glob, WebSearch, WebFetch, Edit, Write

两种截断路径：
- 时间微压缩：使用固定标记替换内容
- 缓存编辑路径：使用 API cache_edits 删除而不破坏缓存

## 历史裁剪

按 API 调用轮次分组，确保工具调用-结果配对完整性。Prompt-Too-Long 时裁剪最老的消息组。
