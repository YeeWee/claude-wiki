WebFetchTool 和 WebSearchTool 构成了 Claude Code 与外部世界交互的关键组件，实现网页内容获取和网络搜索能力。

## 核心要点

1. **WebFetchTool**：从指定 URL 抓取内容，转换为 Markdown 格式，使用 Haiku 模型处理和总结内容
2. **WebSearchTool**：使用 Anthropic API 内置搜索能力获取最新信息
3. **安全机制**：预批准域名列表、域名黑名单检查、跨域重定向限制

## WebFetchTool 架构

### 权限检查流程

1. 预批准域名检查（如 docs.python.org、react.dev 等官方文档站点）
2. 拒绝规则检查
3. 询问规则检查
4. 允许规则检查
5. 默认询问用户

### 核心组件

| 组件 | 说明 |
|------|------|
| Turndown | HTML 转 Markdown，惰性加载（约 1.4MB） |
| LRU 缓存 | 15 分钟 TTL，50MB 最大大小 |
| 域名黑名单 | 通过 Anthropic API 检查 |
| Haiku 模型 | 内容处理和总结 |

### 重定向安全处理

禁止跨域自动跟随重定向。允许的重定向：
- `example.com` → `www.example.com`
- 同域路径变化

禁止的重定向：
- 跨域重定向
- 协议变化（https → http）
- 端口变化
- 包含认证信息的 URL

## WebSearchTool

使用 Anthropic Beta API 的内置搜索能力：
- 最大搜索次数：8
- 支持域名过滤（allowed_domains、blocked_domains）
- 流式进度反馈
- 仅对 firstParty、Vertex AI、Foundry 用户启用

## 资源限制

| 限制项 | 值 |
|--------|-----|
| URL 最大长度 | 2000 字符 |
| HTTP 内容最大长度 | 10MB |
| 请求超时 | 60 秒 |
| 域名检查超时 | 10 秒 |
| 最大重定向次数 | 10 |
