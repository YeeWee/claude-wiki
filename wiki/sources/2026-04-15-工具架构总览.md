# 工具架构总览

工具系统是 Claude Code 的可扩展性基石，通过定义统一的 `Tool` 接口，实现内置工具、MCP 工具、技能工具共享相同接口。

## 核心内容

### Tool 类型接口

泛型接口 `Tool<Input, Output, P>` 定义工具完整生命周期：
- 核心字段：name、aliases、inputSchema、outputSchema、maxResultSizeChars
- 核心方法：call()、description()、prompt()、isEnabled()、checkPermissions()
- UI 渲染方法：renderToolUseMessage、renderToolResultMessage 等

### buildTool 工厂函数

简化工具定义，提供安全默认值：
- `isEnabled`: 默认启用
- `isConcurrencySafe`: 失败关闭（默认不安全）
- `isReadOnly`: 失败关闭（默认假设写操作）
- `checkPermissions`: 委托通用权限系统

### 工具聚合函数

三层架构：
- `getAllBaseTools()`：获取所有内置工具，使用条件展开语法动态加载
- `getTools()`：根据权限上下文返回可用工具，三重过滤机制
- `assembleToolPool()`：合并内置工具和 MCP 工具

### 条件加载机制

通过 Feature Flag、环境变量、用户类型控制工具可用性：
- 静态导入：始终加载的核心工具
- 条件导入：Feature Flag 控制的特性工具
- 惰性加载：破除循环依赖

## 关键文件

| 文件路径 | 主要内容 |
|---------|---------|
| `src/Tool.ts` | Tool 类型定义、buildTool 工厂 |
| `src/tools.ts` | 工具聚合、条件加载 |

## 设计启示

- 安全优先的默认值策略
- 统一接口支持多种工具类型
- 多层过滤机制适应不同运行模式
- 条件导入和惰性加载减少启动开销

## Open Questions
