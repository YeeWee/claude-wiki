Claude Code 的分析系统是一个多层次的数据收集和监控基础设施，采用 OpenTelemetry 作为核心技术框架，通过 GrowthBook 实现动态特性配置。

## 核心模块

| 文件 | 功能 |
|------|------|
| index.ts | 公共 API 入口，事件队列管理 |
| growthbook.ts | GrowthBook 特性开关集成 |
| sink.ts | 分析数据路由逻辑 |
| firstPartyEventLogger.ts | 第一方事件日志系统 |
| firstPartyEventLoggingExporter.ts | OTLP 导出器实现 |
| datadog.ts | Datadog 日志集成 |
| sinkKillswitch.ts | 单 sink 熔断机制 |

## 设计原则

1. 无循环依赖：index.ts 模块没有任何依赖
2. 延迟初始化：事件在 sink 附着前会被队列缓存
3. 隐私保护：元数据类型强制验证
4. 熔断机制：单 sink 熔断允许独立控制导出通道

## GrowthBook 特性开关

关键配置：
- remoteEval: true - 服务端预计算特性值
- cacheKeyAttributes: ['id', 'organizationUUID']
- 定期刷新机制确保长时间运行会话获取最新配置

缓存读取优先级：
1. 环境变量覆盖
2. 配置覆盖
3. 内存 payload 缓存
4. 磁盘缓存

## 事件采样

通过动态配置 `tengu_event_sampling_config` 控制采样率。

事件元数据丰富化：
- core_metadata: 模型、会话、用户类型
- user_metadata: 账户 UUID、组织 UUID
- envContext: 平台、架构、终端类型

## Datadog 集成

基数控制策略：
- MCP 工具名归一化为 'mcp'
- 模型名缩短为短名或 'other'
- 版本号截断保留基础版本

用户分桶估算（30 个桶）保护隐私。

## OpenTelemetry 遥测

三种信号支持：
- Metrics: OTLP, Prometheus, Console
- Logs: OTLP, Console
- Traces: OTLP, Console

OTLP 协议：grpc、http/json、http/protobuf（默认）

## 企业配额管理

配额状态：allowed、allowed_warning、rejected

限制类型：five_hour、seven_day、seven_day_opus、seven_day_sonnet、overage

早期预警机制通过阈值检测提前警告用户。

## Sink 熔断

配置来自 GrowthBook 动态配置 `tengu_frond_boric`，允许独立禁用 Datadog 或第一方事件导出。
