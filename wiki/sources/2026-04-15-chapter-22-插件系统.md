# 第二十二章：插件系统

## 概要

Claude Code 的插件系统是一个强大的扩展机制，允许用户通过安装插件来增强 CLI 的功能。插件可提供自定义命令、AI 代理、技能、钩子、输出样式以及 MCP 服务器集成。

## 核心要点

### 插件与内置技能的区分

- **内置插件（Built-in Plugins）**：随 CLI 发布，用户可通过 `/plugin` UI 启用/禁用，设置持久化到用户配置
- **内置技能（Bundled Skills）**：编译进 CLI 二进制文件，对所有用户可用，用于复杂设置或自动启用逻辑

### 插件标识符格式

使用 `{name}@{marketplace}` 格式作为唯一标识符，内置插件使用 `{name}@builtin` 格式区分。

### 插件目录结构

```
my-plugin/
├── plugin.json          # 清单文件（可选）
├── commands/            # 自定义斜杠命令
├── agents/              # 自定义 AI 代理
├── skills/              # 技能目录
├── hooks/               # 钩子配置
└── .mcp.json            # MCP 服务器配置（可选）
```

### 插件组件类型

```typescript
export type PluginComponent =
  | 'commands'
  | 'agents'
  | 'skills'
  | 'hooks'
  | 'output-styles'
```

### MCP 服务器集成

插件可通过多种方式提供 MCP 服务器：
1. 插件目录中的 `.mcp.json`
2. 清单中的 `mcpServers` 字段（支持 MCPB 文件、JSON 文件路径、内联配置）
3. 多个 MCP 配置源的数组

### 插件钩子系统

插件钩子通过 `loadPluginHooks.ts` 模块加载，支持完整的生命周期钩子事件，并支持热重载（设置变更时自动更新）。

### 错误处理

插件系统使用类型化的错误处理，便于诊断和 UI 展示：
- `path-not-found`、`git-auth-failed`、`network-error`
- `manifest-parse-error`、`manifest-validation-error`
- `mcp-config-invalid`、`hook-load-failed`、`dependency-unsatisfied` 等

## 源文件位置

- `src/plugins/builtinPlugins.ts`：内置插件注册
- `src/plugins/bundled/index.ts`：内置插件初始化
- `src/utils/plugins/schemas.ts`：插件清单 Schema
- `src/utils/plugins/mcpPluginIntegration.ts`：MCP 服务器加载
- `src/utils/plugins/loadPluginHooks.ts`：钩子加载
