LSP（Language Server Protocol）是微软开发的开放协议，用于建立编辑器与语言服务器之间的标准化通信。Claude Code 通过 LSP 实现代码智能功能。

## 核心功能

LSP 在 Claude Code 中扮演"代码理解器"的角色：
- 获取实时代码诊断信息（错误、警告）
- 理解代码结构和语义
- 提供智能代码上下文
- 支持多种编程语言

## 与 MCP 的区别

- MCP: 通用工具扩展
- LSP: 专注于代码智能

## 服务器管理

单例模式管理，使用状态机和代数计数器实现幂等初始化：
- not-started → pending → success/failed

核心状态：
- servers: 服务器名称到实例的映射
- extensionMap: 文件扩展名到服务器名称的映射
- openedFiles: 已打开文件 URI 到服务器名称的映射

## 协议集成

使用 vscode-jsonrpc 实现 JSON-RPC 通信。

初始化握手流程：
1. 发送 initialize 请求
2. 接收 InitializeResult + capabilities
3. 发送 initialized 通知

## 生命周期管理

状态转换规则：
- stopped → starting → running
- running → stopping → stopped
- 任意状态 → error (失败时)
- error → starting (重试时)

崩溃恢复限制防止无限重启（默认 maxRestarts = 3）。

瞬态错误（如 "content modified"）自动重试，指数退避：500ms → 1000ms → 2000ms。

## 诊断系统

异步接收诊断通知，遵循 AsyncHookRegistry 模式：
- MAX_DIAGNOSTICS_PER_FILE: 10
- MAX_TOTAL_DIAGNOSTICS: 30
- MAX_DELIVERED_FILES: 500

LRU 缓存实现跨轮次去重。

## 文件同步

通知类型：
- textDocument/didOpen
- textDocument/didChange
- textDocument/didClose

## 多语言支持

扩展名映射路由请求，插件作用域避免冲突（格式：`plugin:pluginName:name`）。
